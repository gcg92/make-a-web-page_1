Lesson 1: The Basics of the Web and HTML

Web Basics

The wolrd wide web is a colleciton of HTML documents, which is the main document type on the web. In addition to HTML, the following file types can be found on the web:

-Images
-Video
-Plain text
-Music

HTML stands for "Hypertext Markup Language." This is the basis for every web page. HTML documents contain text content (what you see), markup (what it looks like), references to other documents, and links to other pages.This video is a good place to start.

How the Web Works

First, let's define the major pieces of the web:

-Servers - Computers that host the files that make up the web
-Internet - world's largest computer nerwork
-Browser - program that displays files on your computer from the web
-HTTP - the main protocol used by browsers to communicate with servers

The browser on your computer and makes requests to servers via the internet using the HTTP protocol. The servers respond with files that your browser displays. Here's a good explanation.

HTML markup is made up "tags." Tags look something like this: <Tag name> Contents </Tag name>

The two tag names are always the same, while the first one is called the opening tag and the second is called the closing tag. The opening tag, its contents, and the closing tag as a whole can be referred to as an element. HTML tags are used to help the browser identify the type of each element.You can find an introduction of the two here

Computers Are Stupid?

It's important to remember that computers interpret the instructions humans give them literally. One small typo or mistake that "smart" human could correct can lead to huge problems when given to a computer. What do you think happens when you forget to insert a closing tag? Find out here.

Inline vs. Block Elements

You may notice that text on multiple lines will appear on a single line when written in HTML. Tags such as <br> and <p> can aid in creating line breaks. The break tag is an inline element that signifies the end of a line. The paragraph tag is a block element that creates an invisible box around its contents. This video is a big help.

Lesson 2: Creating Structured Documents With HTML

Developer Tools

HTML elements can be either inline or block. Block elements form an "invisible box" around their contents.

The Tree-Like Structure of HTML

First, let's start with some basic definitions:

- HTML - HyperText Markup Language - the standard markup language used to create web pages.

- CSS - Cascading Style Sheets - style sheet language used for describing the look and formatting of a document written in a markup language.

- DOM - Document Object Model - a cross-platform and language-independent convention for representing and interacting with objects in HTML (and other markup languages).

This "tree like" structure comes from the fact that HTML elements can contain other elements inside of them. In a browser, this structure shows up as a series of nested boxes, which can be drawn out like a family tree.

HTML and Boxes

Every HTML element is inside an "invisible box." When you read an HTML document as text, you see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested it's corresponding "box" is.

Text Editors (For Programming)

What is a text editor? A text editor is a program that helps you type text similar to document editors, such as Microsoft Word or Corel WordPerfect, but without all the extra formatting functions that document editors have.

When writing code, programmers use special text editors (like Sublime Text). These make their lives easier by doing things like automatically generating closing HTML tag when you write an opening tag.

Lesson 3: Adding CSS Style To Your HTML Structure

CSS - Search and Replace

CSS stands for "Cascading Style Sheets" and is used to control the style of HTML elements. It can also be thought of as a search and replace tool. First, you identify a tag or class of the element you want to find. Then, you decide what you want to do with it, or what property values to replace with different ones.

Cascading

"Cascading" means that rules are applied not only to the elements they directly match, but also to all of those elements' child elements. However, if a child element has multiple, overlapping rules defined for it, the more specific rule takes effect. This gives programmers an efficient way to control the style of related HTML elements.

Avoiding Repetition

In computer programming, it would be easy to repeat yourself over and over if you didn't know better. This repetition is not only time consuming, but can lead to systemic errors.

Often times you will find yourself writing the same code over and over with only minor differences. For example, how are you going to apply the same style attributes to all elements of a certain type? You could always copy that style and paste it each place that element occurs within your code, but that still leaves a lot of room for error. If you decide to alter the style of a certain element and forget to make that change everywhere, you're in trouble. Remember, computers are stupid.

An easy method for avoiding repetition is through the use of the div element. By assigning similar divs the same class, you can specify the style for that class just once and be done. This also allows you to quickly change the style of that class whenever you want and be sure that the change is made everywhere!

Order Matters

It also matters where you define the rules and in what order they are applied. Styles can be defined in different places and are applied in the following order:

1) the default style of a browser (different browsers have different styles
2) stylesheet in a separate file
3) stylesheet inside HTML
4) inline style in an element - this should be avoided

How to Include CSS On Your Webpage

There are three ways to achieve this:

1) For small projects, you can write CSS in the <head> element of your HTML
2) Link your HTML to a separate CSS file
3) Write your style inline with your HTML, which leads to a lot of repetition

Boxes and Sizing

All HTML elements are boxes made up of four components: content, padding, border, and margin. It can be difficult to get the size of each component just right.

One way to size boxes is by using percentages, where the size of each box changes as the screen size changes. If you want the size of the box to remain unchanged, use pixels. You can also set a max width for each box, so it gets smaller if necessary, but never exceeds a given size.
